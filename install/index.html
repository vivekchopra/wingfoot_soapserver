Installing the Parvus SOAP engine

The first step for installation is to copy the parvus web application from the webapps directory into the appropriate location for your servlet container (eg. Tomcat). Next, we restart the servlet container in order to have the parvus web application loaded.

Now browse over to the default URL http://localhost:8080/parvus/wserver to verify the installation. Here localhost and 8080 are the host and port number that Tomcat runs on- this might be different in your installation.

There are some configuration parameters that may need to be tuned in the wingfoot.properties files. Edit the CATALINA_HOME/webapps/parvus/WEB-INF/classes/wingfoot.properties file (for Tomcat- this maybe be at a different location for your servlet container) and modify as required. The configurable parameters in this file are:

•deploymentFile: This is the file that contains persistent data identifying the services already deployed. Change this to the absolute path of a file. The permissions of the file and directory should be such that only the Tomcat user can read and modify this file.


• logger: This is the file where parvus logs the payload it send/receives. Since the log files are not rolled, set this to a location with some amount of space. The size of the log files should also be monitored periodically. In case this parameter is missing, logging is turned off.

All the paths must be absolute.

Once these parameters have been modified, your servlet container (or the parvus web application) should be restarted for them to take effect.

See the Parvus User Guide for additonal information.
